# Формат CSV для загрузки сертификатов

Импорт через админку: `POST /admin/import` (multipart, файл `file`).

## Обязательные колонки

| Колонка | Описание | Пример |
|---------|----------|--------|
| `denomination_rub` | Номинал в рублях (целое) | `5000` |
| `code` | Номер сертификата (уникальный) | `TEST-SPA-3000-000001` |
| `pin` | PIN-код (если нет — пусто) | `4829` |

## Опциональные колонки

| Колонка | Описание | Пример |
|---------|----------|--------|
| `variant` | `spa` или `ar` (Адмиральский разряд) | `spa` |
| `valid_until` | Срок действия `YYYY-MM-DD` | `2027-12-31` |
| `batch_id` | Метка партии для аудита | `iiko-export-2026-05` |

## Пример файла

См. [`samples/certificates.sample.csv`](../samples/certificates.sample.csv).

## Правила

- Дубликаты `code` в файле и в БД пропускаются, в ответе — счётчики `imported` / `skipped`.
- Номинал должен существовать в справочнике или будет создан автоматически (цена = номинал).
- Коды в статусе `sold` не перезаписываются.

## Экспорт остатков

`GET /admin/stock` — агрегаты по номиналам (без полных номеров).

`GET /admin/stock/export?mask=1` — CSV с маской `****1234` для сверки.

`GET /admin/stock/export?full=1` — полные коды (только с заголовком `X-Export-Full: 1` и записью в audit log).
