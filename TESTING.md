# Инструкция для тестирования MVP (Тестовая СПА продажа)

## Что входит в тестовую сборку

- Покупка подарочного сертификата (Тестовая СПА продажа / Адмиральский разряд)
- Демо-база из 9 тестовых номеров (загружается автоматически при первом запуске)
- Имитация оплаты (без реальных денег)
- Скачивание PDF сертификата после «оплаты» (SMTP не обязателен)
- Админка: остатки, импорт CSV, журнал

---

## Запуск на компьютере тестировщика

### Требования

- Docker Desktop (или Docker + Compose)
- Порты **3000** и **8000** свободны

### Команды

```bash
cd sert_lending
cp .env.test .env
docker compose up -d --build
```

Подождите 1–2 минуты, пока соберутся образы.

### Проверка, что всё поднялось

```bash
curl http://localhost:8000/health
# ожидается: {"status":"ok"}
```

## Сценарий теста (5–10 минут)

### 1. Покупка сертификата

1. Откройте **http://localhost:3000**
2. Вверху жёлтая плашка «Тестовый стенд» — это нормально
3. Выберите тип (Тестовая СПА или Адмиральский разряд) и номинал
4. Заполните имя, email, телефон → **Далее**
5. Отметьте оба согласия → **Перейти к оплате**
6. Страница «Имитация оплаты» → автоматический переход на «Спасибо за покупку»
7. Нажмите **Скачать PDF сертификата** — файл должен открыться

### 2. Повторная покупка

Купите другой номинал — в списке должно уменьшиться «Доступно» (в админке).

### 3. Админка

1. **http://localhost:3000/admin**
2. Логин: `admin`  
   Пароль: `K8test-2026` (из `.env.test`)
3. Проверьте таблицу остатков (available / reserved / sold)
4. Загрузите CSV (можно взять `samples/certificates.sample.csv` с новыми кодами)
5. «Выгрузка (маска)» — коды частично скрыты

### 4. Негативные проверки

| Действие | Ожидание |
|----------|----------|
| «Далее» без телефона/email | Кнопка неактивна или ошибка |
| Оплата без галочек согласия | Кнопка «Перейти к оплате» неактивна |
| Номинал «нет в наличии» | Недоступен в списке (если коды закончились) |

## Отправка стенда тестировщику на VPS

На сервере с Docker:

```bash
git clone <ваш-репозиторий> k8-certificates
cd sert_lending
cp .env.test .env
```

В `.env` замените:

```env
PUBLIC_BASE_URL=http://ВАШ_IP:3000
NEXT_PUBLIC_API_URL=http://ВАШ_IP:8000
CORS_ORIGINS=http://ВАШ_IP:3000,https://spa.k8.ru
```

```bash
docker compose up -d --build
```

Откройте в браузере `http://ВАШ_IP:3000`.

**Важно:** на VPS откройте порты 3000 и 8000 в файрволе.

## Остановка

```bash
docker compose down
```

Данные БД сохраняются в volume. Полный сброс: `docker compose down -v`.

---

## Известные ограничения MVP

- Реальная оплата (ЮKassa) не включена — только `PAYMENT_PROVIDER=mock`
- Email с PDF не уходит без настройки SMTP — используйте кнопку «Скачать PDF»
- Фискальный чек в тесте не формируется
